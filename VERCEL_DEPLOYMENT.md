This guide explains how to deploy your Barbershop application to Vercel with a persistent database (Turso) and working images.

## 1. Prerequisites (Database Setup)
Since Vercel deletes local files on every update, you must use **Turso** to keep your users and appointments permanent.

1.  Create a free database at [Turso.tech](https://turso.tech).
2.  Copy your **Database URL** (`libsql://...`) and **Auth Token**.
3.  Ensure your current code is pushed to GitHub:
    ```bash
    git add .
    git commit -m "Prepare for final deployment"
    git push origin main
    ```

## 2. Vercel Project Configuration
1.  Go to your project on the [Vercel Dashboard](https://vercel.com).
2.  Go to **Settings** > **Environment Variables**.
3.  Add the following **4 Variables**:

| Key | Value | Why? |
| :--- | :--- | :--- |
| `TURSO_URL` | *Your Turso URL* | Connects to the cloud database. |
| `TURSO_AUTH_TOKEN` | *Your Turso Token* | Authenticates the connection. |
| `VITE_API_URL` | `/api` | Tells the frontend where the server is. |
| `JWT_SECRET` | *Any long random string* | Secures user login tokens. |

## 3. Deployment Settings
On the **Settings** > **General** page, verify these:
- **Framework Preset**: Vite
- **Root Directory**: `./` (The root of your repo)
- **Build Command**: `npm run build`
- **Output Directory**: `dist`

## 4. Finalizing Deployment
1.  Go to the **Deployments** tab.
2.  Click the three dots (...) on the latest production deployment and select **Redeploy**.
3.  Once finished, visit your site (e.g., `henibarbershopp.vercel.app`).
